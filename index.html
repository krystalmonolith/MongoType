MongoType

A MongoDB command line tool to dump the contents of a collection and its associated BSON type information.

Usage:

MongoType Command Line Syntax:

mongotype [ <options>] <db>. <collectionName>

Where:

  • <db>.<collection>" is the required name of the MongoDB collection to process as input.

Options:

General Options:
Short
Option		Long
Option		Option
Description
-h--helpprint help message
-v--versionprint version string
-d--debugprint debugging info
-q--stackprint stack debugging info
-c--configpath of configuration file. Default configuration file: ~/.mongotype
MongoDB Server Options:
-h [ --host ] arg (=localhost) MongoDB server host name: [:port] -p [ --port ] arg (=27017) MongoDB server port number.
Output Format Options:
-s [ --style ] arg (=0) Output Style: {dotted,tree,json,jsonpacked}. -t [ --type ] arg (=7) BSON Type: {none,name,desc,code,all}. -f [ --scalarfirst ] arg (=0) Output scalar objects elements before any
embedded objects or arrays.

Compilation & Linking Prerequisites:

  • GNU g++ (gcc v4.7.3 or greater, uses --std=c++0`x )
  • MongoDB C++ Client Library using the "legacy" branch.
  • Boost 1.55.x or greater. This is because of g++ compile option --std=c++0x and issues with TIME_UTC. See MongoDB JIRA Issue SERVER-6340. MongoType will probably compile with Boost 1.50+ with minor backporting.
